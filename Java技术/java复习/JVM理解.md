关于`JDK1.2`以后出现的四种引用，强引用、弱引用、软引用、虚引用

我们一般所引用的对象都是强引用，我们所创建的对象和创建的数，在`JVM`处理这种数据时，当内存不够的时候，`JVM` 就算报错，也不会回收这些内存，

```java
// 强引用
Object obj = new Object();
```

而软引用，就是我们没有给它赋值的数据

```java
// 软引用
System.out.println("hello world");
```

这里是的hello world 就是弱引用，因为他并不是一个变量的值，他只是属于一个临时值，当内存不够用的时候，他就会被 `JVM` 回收

这就相当于电脑的缓存一样，如果电脑的存储空间足够大，不用删除这个缓存，因为在下次加载这些文件的时候，速度会快一点，但是当存储空间不够大的时候，这些缓存所占用的存储空间就会被我们所清空，对磁盘空间进行回收

但是当我们清空之后，下次再运行这个程序的时候，程序就会在内存中从头开始运行编译一遍，相应的数据，也要从头开始重新向内存中写入

弱引用是非必须数据

无论内存空间是否充足，弱引用数据都会被清除

弱引用在Java中需要采用 `WeakReference` 类进行 创建

```java
// 创建一个弱引用String 对象 
WeakReference<String> stringWeakReference = new WeakReference<>(new String("hello world"));
// 输出 这个弱引用的数据
System.out.println(stringWeakReference.get());
// 采用垃圾回收线程，回收一下数据
System.gc();
// 再次输出 弱引用的数据
System.out.println("弱引用中数据"+stringWeakReference.get());
```

这里是输出，我们发现弱引用的数据被回收了

hello world
弱引用中数据null

在Java中一共分为三种数据类型，基本数据类型  引用数据类型    这里是其中String 类型是一个特殊的存在，某种程度上说，它既是引用数据类型，同时也存在基本数据类型。因为在Java虚拟机存在着常量池的概念 ，同时还与字面值有关，与float 类型和 long 类型后面标记关键字有关系。

在Java内存中，yin非全局变量的基本数据类型存在栈中，引用数据类型的地址存在堆中，实际的引用数据存在堆中，这里也就是说对象的实例属性存在堆中。

基本变量类型的传值方式
